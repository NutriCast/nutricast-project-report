NutriCast Project Report

Introduction and Background

NutriCast is a mobile-first nutrition tracking and recommendation platform that leverages artificial intelligence to help users achieve their dietary goals in a personalized and interactive way. Unlike traditional food logging apps, NutriCast allows users to simply upload an image of their meal, from which AI models analyze and return a detailed nutritional breakdown. It further supports users by suggesting meals, tracking macronutrient intake, locating healthy dining options, connecting with friends, and receiving professional advice—all wrapped in an intuitive mobile interface.

The project was developed using a robust technology stack: React Native was chosen for cross-platform frontend development, Spring Boot with Java powered the backend, and MongoDB was used for data persistence. For AI functionality, NutriCast integrated OpenAI’s GPT-4o and Google’s Gemini, both of which significantly accelerated implementation and testing. Additional tools such as Firebase, Google Maps API, GitHub, and Google Cloud Platform supported the app’s infrastructure and deployment.

From a project management perspective, NutriCast was scoped and executed using an agile methodology. The team divided responsibilities by feature set, with each member owning specific modules (e.g., chat, nutrition analysis, dashboards). Weekly standups, frequent code merges, and rigorous testing ensured smooth collaboration and progress tracking. The scope encompassed both functional and social features to deliver a comprehensive wellness platform for mobile users.

In terms of existing solutions, NutriCast draws inspiration from similar health apps like MyFitnessPal but differentiates itself through deeper AI integration and stronger community features. While most apps require manual logging or barcode scanning, NutriCast supports AI-driven meal analysis from images and real-time social engagement, thus enhancing both convenience and user motivation.

Software Technologies Used

NutriCast’s development required a combination of mobile, backend, and AI technologies. On the frontend, React Native was used to ensure that the application could run smoothly on mobile devices with a single codebase. The backend was built using Java with Spring Boot, which provided powerful RESTful APIs and seamless integration with MongoDB.

The MongoDB database was used to store key entities such as users, food logs, friendships, and professional advice posts. For authentication and media storage, Firebase was used due to its reliability and mobile SDK support. The app also utilized the Google Maps API for geolocation features related to restaurant discovery.

In terms of AI, NutriCast relied heavily on GPT-4o for natural language processing and image-based nutrition analysis. Google Gemini was used in testing phases to help generate and validate test cases. Lucidchart and Mermaid.js were used to assist with design diagram generation, saving time during planning stages. Postman and JaCoCo were employed for backend testing and code coverage analysis respectively.

Requirements

Functional and Non-Functional Requirements

NutriCast’s functional requirements included a wide range of user-facing features: account login, photo-based food analysis, nutritional dashboard display, personalized meal suggestions, map-based restaurant search, friend system with messaging, professional advice feed, and barcode scanning. Each of these features was mapped to a core business requirement and supported with backend APIs.

On the non-functional side, the app had to be responsive, secure, and scalable. Real-time updates via Firebase and MongoDB were essential, as was compatibility across mobile platforms. Security and performance were prioritized, especially in handling sensitive user data and processing AI requests efficiently.

Minimum Marketable Features (MMFs)

  • AI Nutrition Analysis: Users can upload meal photos and receive nutritional feedback generated by GPT-4.
  • Nutrition Dashboard: A visual dashboard displays daily and weekly intake of calories, protein, and carbs, powered by MongoDB queries.
  • AI Meal Suggestions: Based on user input or images, GPT-4 suggests healthy meals with nutritional breakdowns.
  • Local Restaurants: Google Maps API shows nearby dining options with customizable filters.
  • Connect with Friends: Users can add friends, view each other's nutrition summaries, and chat.
  • Receive Professional Advice: Verified professionals can post insights and users can view them in a dedicated feed.
  • Barcode Scanner: Users can enter a barcode to retrieve AI-generated nutritional details of food items.

Role of AI in Requirements Implementation

GPT-4o played a significant role in implementing features like image-based meal analysis and intelligent suggestions, while Gemini accelerated the testing of backend endpoints. AI reduced implementation complexity for data models and helped automate tedious tasks like test case generation, leading to increased productivity and fewer bugs. Requirements were identified using a mixture of user stories, feasibility analysis, and impact prioritization, ensuring that AI-intensive features were front-loaded for early validation.

Design

The NutriCast system architecture follows a high-level modular design consisting of four primary layers: frontend, backend, database, and AI services. The frontend is developed using React Native and provides an intuitive mobile interface that communicates with backend REST APIs. These APIs are built using Spring Boot in Java and act as the core application logic layer, handling routing, business logic, and validation. This backend in turn communicates with a MongoDB database, which stores user profiles, food logs, friend connections, and professional posts.

At a higher level, the system design supports horizontal scalability by separating concerns across well-defined services and facilitating independent testing and deployment. The backend services are hosted on Google Cloud Platform, while Firebase handles authentication and media storage, supporting secure, high-performance mobile access. Additionally, the backend can invoke external AI services such as GPT-4o to perform meal analysis and generate responses based on user input.

The architectural design is visually captured in diagrams created using Lucidchart, one of which illustrates the interaction between mobile clients, the backend service, the database, and AI integrations. This clarity helped ensure that every feature could be traced through its stack interactions. The team also utilized Mermaid.js for lightweight component-level diagrams embedded in documentation.

Major design decisions included using React Native for cross-platform support, applying modular controllers and services on the backend, and configuring centralized API request handling through a facade layer in the frontend. These decisions collectively enabled fast feature development and reliable integration of AI services while maintaining code clarity and separation of concerns.

Design Patterns Implementation

Several key design patterns were applied to improve structure and maintainability:

  • Observer Pattern: Used in the frontend via React’s useEffect hooks to listen for state changes and re-render dashboards when user or meal data changed. This ensured reactivity without requiring explicit data fetches.
  • Facade Pattern: A centralized api wrapped axios configuration for all backend calls. This simplified each component’s data-fetching logic and allowed global headers to be managed in one place.
  • Singleton Pattern: Firebase was initialized as a singleton to avoid multiple app instances. This was critical for maintaining a single point of truth for authentication and storage.

These patterns helped reduce code duplication, simplified component design, and made debugging significantly easier. Their application also made it easier to onboard new developers or adjust architecture mid-project.

Testing Strategy, Execution, and AI Analysis

Black-Box Testing

Postman was used for black-box testing of backend endpoints. A total of 15 test cases were created for the FoodLogController, testing both positive and negative scenarios. Sampling methods such as equivalence partitioning and boundary value analysis helped ensure robustness. Notable tests included logging meals with minimum servings, requesting data for invalid users, and missing required headers.

Gemini was used during this phase to generate test cases automatically. This AI-enhanced method identified additional edge cases and significantly sped up the testing process.

White-Box Testing

For white-box testing, both manually written and AI-generated unit tests were applied. GPT-4o was used to produce tests targeting models and controllers. Compared to manual efforts, AI-generated tests required less time (5–10 minutes vs. 30–45 minutes) and achieved slightly higher code coverage (74% vs. 68%). When both methods were combined, the project achieved a final code coverage of 83%, as verified using JaCoCo, allowing us to conclude that a hybrid of manual and AI-generated tests was the best approach.

AI Performance and Usability

In testing, AI tools contributed to faster feedback loops, higher code quality, and broader test case coverage. The productivity gain from using GPT-4o and Gemini was notable, especially for repetitive or pattern-based tasks. AI-supported tools were found to be highly effective for black-box validation and white-box modeling tasks.

Challenges and Innovations

Several key challenges shaped the development process and spurred innovation:

  • Image Analysis: Integrating image-based nutritional analysis required uploading to Firebase, generating prompts for GPT-4o, and parsing JSON outputs. This process required multiple iterations to ensure accuracy.
  • Map Integration: The team switched from Mapbox to Google Maps API due to integration bugs, leveraging the Nearby Places API and AI filtering to display relevant restaurants.
  • React Native Components: Web elements like <div> needed to be replaced with native components like <View>, and issues like keyboard overlap required UI refactoring, leading to a stronger mobile-first mindset.

Project Outcomes and Evaluation

NutriCast was successfully deployed as a fully functional mobile application. Users can log meals, analyze food (including barcodes), view dashboards, receive meal suggestions, explore nearby restaurants, connect with friends, and view professional posts.

AI tools played a pivotal role in achieving these outcomes. GPT-4o was central to features like meal analysis and suggestions, while Gemini improved testing and robustness. Overall, AI-support accelerated development by a conservative 30%.

From a quantitative standpoint, the project reached 83% code coverage, passed all critical test cases, and maintained low bug rates during testing. Qualitatively, the application was well-received in demos, showcasing a fluid user experience and reliable functionality.

Future Directions

There are several potential areas for future development. One direction involves integrating a calendar feature for meal planning. Another potential expansion could include exercise tracking and burned calories for a more comprehensive wellness application.

AI tools could also be improved through better prompt tuning, fallback mechanisms for failed responses, and potentially incorporating alternative LLMs for comparison. Future research could focus on user retention and behavior change outcomes in AI-driven nutrition apps compared to traditional methods.
